## 🔴 Critical

### C1 — slice-receipt status enum mismatch with producer

**File**: `schemas/payload-registry/deep-work/slice-receipt/v1.0.schema.json:32-35`

**Convergence**: 2/3 reviewers (Codex review P2 + Codex adversarial HIGH).

**Evidence (pinned producer source)**:
`claude-deep-work@6f23e79.../agents/implement-slice-worker.md` documents the canonical status values:
- `"status": "complete"` — slice succeeded (default success path)
- `"status": "blocked"` — slice failed after 3 consecutive attempts; writes `debug.root_cause_note` to payload
- `"status": "blocked-upstream"` — placeholder for not-yet-started slices when an earlier slice is `blocked`

**Suite schema (Phase 3 PR)**: `enum: ["complete", "partial", "failed", "passed", "skipped"]`.

**Mismatch**: Producer emits `blocked` and `blocked-upstream` on failure paths; suite enum rejects both. Producer never emits `partial`/`failed`/`passed`/`skipped` per the documented contract.

**Impact**: `validate-artifact --strict` against any blocked deep-work slice receipt fails payload schema check, blocking M4 telemetry / dashboard aggregation on exactly the diagnostic-critical artifacts (failure receipts).

**Fix**: Replace enum with producer-documented values.
```json
"status": { "enum": ["complete", "blocked", "blocked-upstream"] }
```

## 🟡 Warning

### W1 — deep-docs payload.provenance rejects optional `path_check_enabled`

**File**: `schemas/payload-registry/deep-docs/last-scan/v1.0.schema.json:13-30`

**Convergence**: 1/3 (Codex adversarial MEDIUM).

**Evidence (pinned producer source)**:
`claude-deep-docs@3cc5229.../agents/doc-scanner.md` documents the optional payload provenance field:
> `payload.provenance.path_check_enabled` (bool, **optional**) — `scan-filters/cli-whitelist.md` 의 `$PATH` 체크가 ON 일 때만 emit (`true`). OFF 일 때는 omit. 재사용 4-요소 규칙의 `prov.get("path_check_enabled", False) != bool(config.enable_path_check)` 비교에 사용.

**Suite schema (Phase 3 PR)**: `additionalProperties: false` + `required: ["is_git"]` + `properties: {is_git, worktree_hash}`. No `path_check_enabled`.

**Mismatch**: When users enable PATH check, producer emits `provenance: { is_git, worktree_hash, path_check_enabled: true }` which fails `additionalProperties` check.

**Fix**: Declare `path_check_enabled` as optional boolean.
```json
"properties": {
  "is_git": { ... },
  "worktree_hash": { ... },
  "path_check_enabled": { "type": "boolean", "description": "Emitted only when scan-filters/cli-whitelist.md $PATH check is ON." }
}
```

### W2 — dashboard dimensions array lacks exact-6 enforcement

**File**: `schemas/payload-registry/deep-dashboard/harnessability-report/v1.0.schema.json:32-33`

**Convergence**: 1/3 (Codex review P2).

**Evidence (pinned producer source)**:
`claude-deep-dashboard@cfd07bd.../scripts/validate-envelope-emit.js`:
```js
// Mirror the producer-side invariant (lib/harnessability/scorer.js + checklist.json):
// exactly 6 dimensions are emitted (type_safety / module_boundaries / test_infra /
// sensor_readiness / linter_formatter / ci_cd).
if (pl.dimensions.length !== 6) {
  fail(`payload.dimensions must have exactly 6 entries (got ${pl.dimensions.length})`);
}
```

**Suite schema (Phase 3 PR)**: `minItems: 1`.

**Mismatch**: Producer enforces exactly 6, suite allows 1–∞. Truncated reports pass suite-side strict validation but fail producer-side validation — divergence point that M4 telemetry can't surface.

**Fix**: Add `maxItems: 6` and bump `minItems: 1 → 6`.
```json
"dimensions": { "type": "array", "minItems": 6, "maxItems": 6, ... }
```
